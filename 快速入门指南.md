# 自动开发服务器启动工具 - 快速入门指南

欢迎使用自动开发服务器启动工具！本指南将帮助您在 5 分钟内上手使用。

---

## 📦 安装

### 方式一：安装程序（推荐）

1. 双击运行 `自动开发服务器启动工具 Setup 1.0.1.exe`
2. 按照安装向导完成安装
3. 从桌面快捷方式启动应用

### 方式二：便携版

1. 解压 `win-unpacked` 文件夹到任意位置
2. 双击运行 `自动开发服务器启动工具.exe`

---

## 🚀 第一次使用

### 步骤 1: 添加项目

1. 点击界面顶部的 **"📁 选择项目文件夹"** 按钮
2. 在弹出的对话框中，选择您的项目文件夹
3. 项目会自动添加到左侧列表

> **提示**: 项目文件夹必须包含 `package.json` 文件

### 步骤 2: 启动项目

1. 在左侧列表中点击您的项目
2. 点击 **"▶️ 启动"** 按钮
3. 右侧会显示服务器输出

> **自动安装**: 如果项目缺少 `node_modules`，应用会自动安装依赖！

### 步骤 3: 查看输出

- 右侧面板实时显示服务器输出
- 输出中的 URL 可以直接点击打开
- 状态徽章显示项目运行状态

### 步骤 4: 停止项目

1. 选中正在运行的项目
2. 点击 **"⏹️ 停止"** 按钮

## 💡 核心功能

### 🔄 自动依赖安装

**无需手动 npm install！**

当您启动一个没有 `node_modules` 的项目时：
1. 应用自动检测包管理器（pnpm/yarn/npm）
2. 自动执行安装命令
3. 实时显示安装进度
4. 安装完成后自动启动

**示例输出**:
```
检测到缺少 node_modules 目录，正在使用 pnpm 安装依赖...
执行命令: pnpm install

Packages: +399
Progress: resolved 399, reused 399, downloaded 0, added 399, done

✓ 依赖安装完成，开始启动项目...
```

### 🛡️ 智能进程管理

**自动清理，无后顾之忧！**

- **删除项目**: 自动停止正在运行的服务器
- **关闭应用**: 自动停止所有服务器进程
- **无孤立进程**: 不会留下占用端口的后台进程

## 🎯 常见使用场景

### 场景 1: 新克隆的项目

```
1. 从 Git 克隆项目到本地
2. 在应用中选择项目文件夹
3. 点击"启动"
4. 应用自动安装依赖并启动 ✨

```

```
1. 添加前端项目并启动
2. 添加后端项目并启动
3. 两个项目同时运行
4. 在不同项目间切换查看输出
```

```
1. 手动删除 node_modules（节省空间）
2. 在应用中点击"启动"
3. 应用自动重新安装依赖 ✨
4. 项目正常启动
```

## 📊 界面说明

### 左侧 - 项目列表

显示所有已添加的项目，包括：
- **项目名称**: 从 package.json 读取
- **项目路径**: 完整的文件夹路径
- **最后启动时间**: 上次启动的时间
- **状态徽章**: 当前运行状态

**状态说明**:
- 🔵 **未启动** - 灰色
- 🟡 **启动中** - 金色
- 🟢 **运行中** - 绿色
- ⚫ **已停止** - 灰色
- 🔴 **错误** - 橙色

### 右侧 - 服务器输出

实时显示选中项目的输出，包括：
- 依赖安装进度
- 服务器启动信息
- 访问地址（可点击）
- 错误信息

### 操作按钮

- **▶️ 启动**: 启动选中的项目
- **⏹️ 停止**: 停止正在运行的项目
- **📂 打开**: 在文件管理器中打开项目文件夹
- **🗑️ 删除**: 从历史记录中删除（不会删除文件）

## ⚠️ 注意事项

### 必需条件

1. ✅ 项目文件夹必须包含 `package.json`
2. ✅ package.json 中必须有启动脚本（dev/start/serve）
3. ✅ 系统已安装 Node.js 和包管理器

### 首次使用

1. **依赖安装时间**: 首次安装可能需要 1-5 分钟
2. **网络连接**: 需要稳定的网络下载依赖包
3. **磁盘空间**: 确保有足够空间存储依赖

### 最佳实践

1. **保存工作**: 关闭应用前保存所有未保存的工作
2. **查看输出**: 遇到问题时查看右侧输出面板
3. **多项目管理**: 可以同时运行多个项目

## 🔧 高级技巧

### 自定义配置（可选）

如果需要更精细的控制，可以在项目根目录创建 `dev-config.json`:

```json
{
  "name": "我的项目",
  "command": "pnpm run dev",
  "cwd": "/path/to/project",
  "env": {
    "NODE_ENV": "development",
    "PORT": "3000"
  }
}
```

### 包管理器检测规则

应用按以下优先级检测包管理器：
1. 存在 `pnpm-lock.yaml` → 使用 **pnpm**
2. 存在 `yarn.lock` → 使用 **yarn**
3. 存在 `package-lock.json` → 使用 **npm**
4. 默认使用 **npm**

## ❓ 常见问题

### Q: 为什么提示"未找到 package.json"？
**A**: 请确保选择的是项目根目录，且包含 `package.json` 文件。

### Q: 依赖安装失败怎么办？
**A**: 
1. 检查网络连接
2. 查看输出面板的错误信息
3. 尝试手动安装依赖后再启动

### Q: 可以同时启动多个项目吗？
**A**: 可以！每个项目都是独立的进程，互不影响。

### Q: 关闭应用后服务器会停止吗？
**A**: 会的。应用会自动停止所有服务器，不会留下后台进程。

### Q: 如何查看完整的错误信息？
**A**: 所有输出都会显示在右侧面板中，包括详细的错误堆栈。
